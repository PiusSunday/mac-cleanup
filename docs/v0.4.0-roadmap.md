# mac-cleanup vNext: Complete Architecture Analysis & v0.4.0 Roadmap

## PART I — CRITICAL FINDINGS (Stability & Security Audit)

### Finding 1 — `caches::_user_caches` is a Loose Cannon (HIGH)

**File**: `mac-cleanup/lib/caches.sh`

```bash
# CURRENT — DANGEROUS
dry_run_or_exec rm -rf "$cache_dir"   # deletes EVERY subdir of ~/Library/Caches
```

**Problem**: This is a blind `rm -rf` on every subdirectory of `~/Library/Caches` with only a running-process check and the tiny `SIP_PROTECTED_PATHS` list as guards. If the process name in `pgrep -x` doesn't exactly match the cache dir name (e.g., `"Slack Helper (Renderer)"` vs `"Slack"`), the guard silently passes. Missing from the protected list: `com.apple.bird`, `com.apple.nsurlsessiond`, `com.apple.WebKit.Networking`, any `CloudKit` subfolder, Font registry caches, Spotlight indexes.

**Mole's approach**: Has `DEFAULT_WHITELIST_PATTERNS` with explicit Apple system caches, checks `should_protect_path`, and uses `safe_clean` which validates before every delete. We have no equivalent hardened whitelist.

**Impact if triggered**: Clearing `com.apple.Spotlight` or `com.apple.FontRegistry` causes Spotlight to re-index the entire disk, font rendering to break, and system search to fail — hours of recovery.

---

### Finding 2 — `caches::_user_logs` deletes the entire log directory (HIGH)

**File**: `mac-cleanup/lib/caches.sh`

```bash
dry_run_or_exec rm -rf "$path"     # "$path" = ~/Library/Logs (the DIRECTORY itself)
```

**Problem**: We delete the _directory_, not its contents. On macOS, `~/Library/Logs` is a SIP-managed metadata boundary — deleting the directory root leaves orphaned processes that expect it to exist and can cause crash-loop behavior until the next login/restart. The `is_deletable` check passes because the parent `~/Library` is writable.

**Fix**: Target `"$path"/*` (contents), never the container directory.

---

### Finding 3 — `dry_run_or_exec` has a Dry-Run Accuracy Gap (MEDIUM)

**File**: `mac-cleanup/lib/utils.sh`

```bash
dry_run_or_exec() {
  # ...
  if [[ "$DRY_RUN" == "true" ]]; then
    log::info "[DRY-RUN] Would execute: ${DIM}${pretty_cmd}${RESET}"
    return 0  # ← returns without measuring the path
  fi
```

**Problem**: Dry-run mode logs what _command_ would run but never calculates what _bytes_ would be freed. The summary report therefore shows 0 freed in dry-run mode, making it useless for capacity planning. A user running `--dry-run` before a big cleanup gets no meaningful output.

**Mole's approach**: `--dry-run` mode walks the same logic path, calculates sizes with `get_path_size_kb`, and prints `"would clean XMB"` before each item.

---

### Finding 4 — `devtools::_build_exclude_args` uses `eval` with unsanitized paths (MEDIUM)

**File**: `mac-cleanup/lib/devtools.sh`

```bash
eval "$assign_str"    # builds an array via eval from user-influenced path values
```

**Problem**: `DEVTOOLS_SCAN_DIRS` and `DEVTOOLS_EXCLUDE_PATHS` are composed from `$HOME`. If `$HOME` contains shell metacharacters (unlikely but theoretically possible on adversarially configured systems), `eval` expands them. Beyond the immediate risk, `eval` is a code smell that makes the logic fragile and harder to test.

**Fix**: Replace `eval`-based array building with a `nameref` (`local -n`) or a direct accumulator, both available in bash 4.3+ (macOS 12+ ships bash 3, but `#!/usr/bin/env bash` picks up Homebrew's bash 5).

---

### Finding 5 — No Pre-Flight Safety Gate (HIGH)

**Current state**: The tool runs cleanup immediately after parsing flags. There is no check for:

- Available disk space before deleting (if `$HOME` is near full, deleting caches while Spotlight is writing may corrupt the index)
- Battery level (a delete interrupted by forced shutdown mid-`rm -rf` leaves partial state)
- SIP status (crucial for knowing what will or won't work without surfacing confusing errors)
- Active Time Machine backup in progress (deleting files during a TM snapshot causes it to checkpoint incorrectly)

**Mole has**: `check_filevault`, `check_firewall`, `check_gatekeeper`, battery checks, and an explicit pre-flight stage.

---

### Finding 6 — `system::_system_logs` is Read-Only Theater (LOW)

**File**: `mac-cleanup/lib/system.sh`

```bash
log::info "System logs: ${size_fmt} (read-only report — cleaning via this tool is not supported)"
```

We measure `/private/var/log`, announce the size, and do nothing. This wastes a `du` call and confuses users who see `"2.3 GB"` reported but nothing cleaned.

---

### Finding 7 — `caches::_containers` skips age-gating (MEDIUM)

**File**: `mac-cleanup/lib/caches.sh`
The containers cleanup deletes entire `~/Library/Containers/<bundle>` directories for apps that aren't running. Sandboxed App Store apps store **user data** (not just caches) inside `~/Library/Containers`. A stopped app (e.g., Fantastical not currently open) would have its entire container deleted, including its SQLite database.

**Fix**: Only target `Cache` and `tmp` subdirectories _within_ containers, never the container root.

## PART IV — THE v0.4.0 SUPERCHARGE PLAN

### A. Core Architecture Refactor

#### A1 — Hardened Delete Primitive

Replace every raw `rm -rf` with a single battle-tested function. This is the single most impactful change:

```bash
# lib/utils.sh — NEW: safe_rm()
# The ONLY function that may call rm. Everything else calls this.
safe_rm() {
  local path="$1"
  local label="${2:-$path}"

  # 1. Must be absolute
  [[ "$path" == /* ]] || { log::warn "safe_rm: relative path rejected: $path"; return 1; }

  # 2. Path traversal guard
  if [[ "$path" =~ (^|/)\.\.(/|$) ]]; then
    log::error "safe_rm: path traversal rejected: $path"; return 1
  fi

  # 3. Symlink target validation — resolve and re-check
  if [[ -L "$path" ]]; then
    local target
    target=$(readlink -f "$path" 2>/dev/null) || { log::warn "safe_rm: unresolvable symlink: $path"; return 1; }
    _safe_rm_check_system_path "$target" || return 1
  fi

  # 4. System path check (covers /System, /bin, /usr, /private/etc, etc.)
  _safe_rm_check_system_path "$path" || return 1

  # 5. Whitelist check
  utils::is_whitelisted "$path" && { log::verbose "safe_rm: whitelisted, skipping: $path"; return 0; }

  # 6. Parent write permission
  [[ -w "$(dirname "$path")" ]] || { log::verbose "safe_rm: no write permission: $path"; return 1; }

  # 7. Size measurement for reporting (always, even in dry-run)
  local bytes=0
  [[ -e "$path" ]] && bytes=$(utils::get_size_bytes "$path")

  if [[ "$DRY_RUN" == "true" ]]; then
    log::info "  [DRY-RUN] $(utils::format_bytes "$bytes")  ${label}"
    # Accumulate dry-run total for accurate summary
    TOTAL_DRYRUN_BYTES=$(( TOTAL_DRYRUN_BYTES + bytes ))
    return 0
  fi

  # 8. Execute
  rm -rf -- "$path" 2>/dev/null || {
    log::verbose "safe_rm: deletion failed (permission): $path"
    return 1
  }

  # 9. Log to operation log
  _oplog "REMOVED" "$path" "$(utils::format_bytes "$bytes")"
  TOTAL_FREED=$(( TOTAL_FREED + bytes ))
  log::success "  $(utils::format_bytes "$bytes")  ${label}"
}

_safe_rm_check_system_path() {
  local p="$1"
  case "$p" in
    /|/System|/System/*|/bin|/bin/*|/sbin|/sbin/*|/usr|/usr/bin/*|/usr/lib/*|\
    /etc|/etc/*|/private/etc|/private/etc/*|/Library/Extensions|\
    /Library/Extensions/*|/private/var/db/receipts)
      log::error "safe_rm: system path rejected: $p"; return 1 ;;
  esac
  return 0
}
```

#### A2 — User-Configurable Whitelist System

```bash
# lib/utils.sh — NEW: whitelist loader
WHITELIST_FILE="$HOME/.config/mac-cleanup/whitelist"

_whitelist_patterns=()

utils::load_whitelist() {
  # Built-in Apple system caches — never delete
  _whitelist_patterns=(
    "$HOME/Library/Caches/com.apple.FontRegistry"
    "$HOME/Library/Caches/com.apple.Spotlight"
    "$HOME/Library/Caches/com.apple.spotlight"
    "$HOME/Library/Caches/CloudKit"
    "$HOME/Library/Caches/com.apple.finder"
    "$HOME/Library/Mobile Documents"
    "$HOME/.ollama/models"
    "$HOME/.cache/huggingface"
  )
  # User overrides
  if [[ -f "$WHITELIST_FILE" ]]; then
    while IFS= read -r line; do
      [[ -z "$line" || "$line" == \#* ]] && continue
      line="${line/#\~/$HOME}"
      _whitelist_patterns+=("$line")
    done < "$WHITELIST_FILE"
  fi
}

utils::is_whitelisted() {
  local path="$1"
  for pattern in "${_whitelist_patterns[@]}"; do
    [[ "$path" == "$pattern"* ]] && return 0
  done
  return 1
}
```

#### A3 — Operation Log (Undo Trail)

```bash
# lib/utils.sh — NEW: operation log
_OPLOG="$HOME/.mac-cleanup/operations.log"

_oplog() {
  local action="$1"  # REMOVED | SKIPPED | DRYRUN
  local path="$2"
  local size="$3"
  mkdir -p "$(dirname "$_OPLOG")"
  printf "[%s] [%s] %s  (%s)\n" \
    "$(date '+%Y-%m-%dT%H:%M:%S')" "$action" "$path" "$size" >> "$_OPLOG"
}
```

This gives users a complete audit trail. A future `mac-cleanup --show-log` command reads it. A `--quarantine` flag (Phase 2) could move files to `~/.Trash` instead of `rm`, making every delete recoverable.

#### A4 — Pre-Flight Check Module (`lib/preflight.sh`)

```bash
#!/usr/bin/env bash
# lib/preflight.sh — Safety gates that must pass before any deletion

preflight::run() {
  log::section "Pre-Flight Checks"

  local pass=true

  # 1. Free disk space — warn if <5 GB
  local free_bytes
  free_bytes=$(utils::get_free_bytes)
  local free_gb=$(( free_bytes / 1073741824 ))
  if (( free_gb < 5 )); then
    log::warn "Low disk space: ${free_gb} GB free. Cleanup may be unstable."
    if [[ "$SKIP_CONFIRM" != "true" ]]; then
      utils::confirm "Continue with low disk space?" || exit 0
    fi
  else
    log::success "Disk space: ${free_gb} GB free"
  fi

  # 2. Time Machine — warn if backup in progress
  if tmutil status 2>/dev/null | grep -q '"Running" = 1'; then
    log::warn "Time Machine backup is in progress. Deletion during a backup may cause inconsistencies."
    utils::confirm "Continue while Time Machine is running?" || exit 0
  fi

  # 3. Battery — warn on laptops with <20% and not on AC power
  if command -v pmset &>/dev/null; then
    local battery_info
    battery_info=$(pmset -g batt 2>/dev/null || true)
    local battery_pct
    battery_pct=$(echo "$battery_info" | grep -oE '[0-9]+%' | head -1 | tr -d '%')
    local on_ac
    on_ac=$(echo "$battery_info" | grep -c "AC Power" || true)
    if [[ -n "$battery_pct" && "$on_ac" -eq 0 && "$battery_pct" -lt 20 ]]; then
      log::warn "Battery at ${battery_pct}% — not on AC power. A forced shutdown could cause data loss."
      utils::confirm "Continue on low battery?" || exit 0
    fi
  fi

  # 4. SIP status (informational)
  if command -v csrutil &>/dev/null; then
    if csrutil status 2>/dev/null | grep -q "enabled"; then
      log::success "SIP: enabled (system paths are protected)"
    else
      log::warn "SIP: DISABLED — extra caution applied to system paths"
    fi
  fi

  [[ "$pass" == "true" ]]
}
```

### B. New Deep-Clean Modules

#### B1 — `lib/orphans.sh` — Orphaned App Data Detector

This is the biggest gap between us and Mole, and the biggest opportunity to surpass both.

```bash
#!/usr/bin/env bash
# lib/orphans.sh — Detect ~/Library/Application Support, Containers, and
#                  Preferences entries for apps no longer installed.

orphans::clean() {
  log::section "Orphaned App Data"

  # Build set of installed bundle IDs
  local -A installed_ids=()
  _orphans_scan_installed installed_ids

  # Scan targets
  orphans::_app_support "$installed_ids"
  orphans::_containers  "$installed_ids"
  orphans::_preferences "$installed_ids"
}

_orphans_scan_installed() {
  local -n _ids="$1"
  local -a app_dirs=(
    "/Applications"
    "/System/Applications"
    "$HOME/Applications"
    "/opt/homebrew/Caskroom"
  )
  for dir in "${app_dirs[@]}"; do
    [[ -d "$dir" ]] || continue
    while IFS= read -r plist; do
      local bid
      bid=$(/usr/libexec/PlistBuddy -c "Print :CFBundleIdentifier" "$plist" 2>/dev/null) || continue
      [[ -n "$bid" ]] && _ids["$bid"]=1
    done < <(find "$dir" -name "Info.plist" -maxdepth 4 -type f 2>/dev/null)
  done
}

# Normalize: "com.adobe.Photoshop-2025" → "com.adobe.photoshop"
_orphans_normalize_bundle() {
  echo "${1,,}" | sed 's/[-_][0-9].*$//'
}

orphans::_app_support() {
  local -n _ids="$1"
  local base="$HOME/Library/Application Support"
  [[ -d "$base" ]] || return 0

  log::info "Scanning Application Support for orphaned entries..."
  while IFS= read -r dir; do
    local name
    name=$(basename "$dir")
    # Skip known system entries
    case "$name" in
      Apple|com.apple.*|MobileSync|Xcode|JetBrains|\
      Caches|SyncServices|CallHistoryDB|com.microsoft.*) continue ;;
    esac
    # Skip if any installed bundle ID matches (prefix match, case-insensitive)
    local norm_name
    norm_name=$(_orphans_normalize_bundle "$name")
    local matched=false
    for bid in "${!_ids[@]}"; do
      local norm_bid
      norm_bid=$(_orphans_normalize_bundle "$bid")
      if [[ "$norm_bid" == *"$norm_name"* || "$norm_name" == *"$norm_bid"* ]]; then
        matched=true; break
      fi
    done

    [[ "$matched" == "true" ]] && continue

    # Only report entries not modified in ORPHAN_AGE days
    local last_mod
    last_mod=$(find "$dir" -maxdepth 0 -newer /tmp/.mac-cleanup-epoch 2>/dev/null | wc -l)
    local size_bytes
    size_bytes=$(utils::get_size_bytes "$dir")
    local size_fmt
    size_fmt=$(utils::format_bytes "$size_bytes")

    log::warn "  Orphaned?: ${name} (${size_fmt}) — no matching installed app"
    # Orphans are NEVER auto-deleted — always require explicit --clean-orphans flag
    _ORPHAN_REPORT+=("$dir|$size_fmt|$name")
  done < <(find "$base" -mindepth 1 -maxdepth 1 -type d 2>/dev/null)
}
```

**Key design decision**: Orphan _detection_ runs automatically. Orphan _deletion_ requires `--clean-orphans` and interactive per-item confirmation. This is where we surpass Mole: Mole auto-deletes orphans after 30 days; we present a human-readable report and require explicit choices.

#### B2 — `lib/system_deep.sh` — Deep System Rot Cleanup

These are the paths that accumulate over years and standard tools (including Mole's user-mode) miss:

```bash
#!/usr/bin/env bash
# lib/system_deep.sh — Age-gated system rot cleanup (requires sudo for some paths)

# Age thresholds
readonly DEEP_LOG_AGE_DAYS=30
readonly DEEP_TMP_AGE_DAYS=7
readonly DEEP_CRASH_AGE_DAYS=14

system_deep::clean() {
  log::section "Deep System Cleanup"

  system_deep::_unified_logs          # /private/var/db/diagnostics/*.tracev3
  system_deep::_power_logs            # /private/var/db/powerlog
  system_deep::_memory_exception_reports
  system_deep::_var_log_rotated       # /private/var/log/*.gz, *.asl
  system_deep::_private_tmp           # /private/tmp (age-gated)
  system_deep::_os_installer_leftovers
  system_deep::_broken_preferences    # ~/Library/Preferences/*.plist via plutil -lint
  system_deep::_safari_content_cache  # only fsCachedData, not full Safari cache
}

system_deep::_unified_logs() {
  local base="/private/var/db/diagnostics"
  if ! sudo test -d "$base" 2>/dev/null; then
    log::verbose "Unified log dir not accessible: $base"
    return 0
  fi

  log::info "Scanning Unified Log archives (>30 days)..."
  local count=0 total_bytes=0
  while IFS= read -r f; do
    local fb
    fb=$(utils::get_size_bytes "$f")
    total_bytes=$(( total_bytes + fb ))
    (( count++ )) || true
    if [[ "$DRY_RUN" != "true" ]]; then
      sudo rm -f -- "$f" 2>/dev/null || true
      _oplog "REMOVED" "$f" "$(utils::format_bytes "$fb")"
    else
      log::info "  [DRY-RUN] $(utils::format_bytes "$fb")  $f"
    fi
  done < <(sudo find "$base" -type f \( -name "*.tracev3" -o -name "*.logdata" \) \
    -mtime "+${DEEP_LOG_AGE_DAYS}" 2>/dev/null || true)

  (( count > 0 )) && log::success "Unified Log archives: ${count} files ($(utils::format_bytes "$total_bytes"))"
}

system_deep::_broken_preferences() {
  local prefs="$HOME/Library/Preferences"
  [[ -d "$prefs" ]] || return 0
  log::info "Scanning for corrupted preferences..."
  local count=0
  while IFS= read -r plist; do
    local name
    name=$(basename "$plist")
    # Never touch Apple system prefs
    case "$name" in
      com.apple.*|.GlobalPreferences*|loginwindow.plist) continue ;;
    esac
    plutil -lint "$plist" &>/dev/null && continue
    log::warn "  Corrupted plist: $name"
    safe_rm "$plist" "Corrupted pref: $name"
    (( count++ )) || true
  done < <(find "$prefs" -maxdepth 1 -name "*.plist" -type f 2>/dev/null)
  (( count > 0 )) && log::success "Corrupted preferences removed: $count"
}

system_deep::_os_installer_leftovers() {
  # macOS installer app bundles older than 14 days
  local cleaned=0
  for installer in "/Applications/Install macOS"*.app; do
    [[ -d "$installer" ]] || continue
    pgrep -f "$installer" &>/dev/null && continue  # running
    local age_days
    age_days=$(( ( $(date +%s) - $(stat -f%m "$installer") ) / 86400 ))
    (( age_days < 14 )) && continue
    local size_bytes
    size_bytes=$(utils::get_size_bytes "$installer")
    log::warn "  macOS installer (${age_days} days old): $(utils::format_bytes "$size_bytes")"
    safe_rm "$installer" "Old macOS installer"
    (( cleaned++ )) || true
  done
  # /macOS Install Data (12 GB leftover)
  if [[ -d "/macOS Install Data" ]]; then
    local age_days=$(( ( $(date +%s) - $(stat -f%m "/macOS Install Data") ) / 86400 ))
    if (( age_days >= 14 )); then
      local size_bytes
      size_bytes=$(utils::get_size_bytes "/macOS Install Data")
      log::warn "  /macOS Install Data (${age_days} days old, $(utils::format_bytes "$size_bytes"))"
      if sudo test -d "/macOS Install Data"; then
        sudo rm -rf -- "/macOS Install Data" 2>/dev/null && \
          _oplog "REMOVED" "/macOS Install Data" "$(utils::format_bytes "$size_bytes")" || true
      fi
    fi
  fi
}
```

```bash
# lib/devops_reset.sh — Complete developer environment artifact purge
# Invoked only via: mac-cleanup --devops-reset
# This module intentionally goes further than the default --devtools flag.

devops_reset::run() {
  log::section "DevOps Reset Mode"
  log::warn "This will purge ALL development tool caches. This is irreversible."
  utils::confirm "Proceed with DevOps Reset?" || return 0

  devops_reset::_docker_full     # images, stopped containers, volumes (explicit list)
  devops_reset::_node_deep       # npm, pnpm, yarn, bun, tnpm — all caches + global modules
  devops_reset::_python_deep     # pip, poetry, conda, pyenv, venv dirs in projects
  devops_reset::_ruby_deep       # gem cache, bundler standalone
  devops_reset::_java_deep       # gradle, maven snapshot artifacts only
  devops_reset::_rust_deep       # cargo registry, rustup old toolchains
  devops_reset::_ml_frameworks   # torch, tensorflow, huggingface (with whitelist option)
}

devops_reset::_docker_full() {
  utils::require docker || return 0
  if ! docker info &>/dev/null 2>&1; then
    log::info "Docker daemon not running — skipping."
    return 0
  fi

  log::info "Docker: enumerating objects for removal..."

  # NEVER use docker system prune — enumerate and confirm each category
  local -a stopped_containers
  mapfile -t stopped_containers < <(docker ps -a --filter status=exited --format '{{.ID}} {{.Names}} {{.Size}}' 2>/dev/null)
  if (( ${#stopped_containers[@]} > 0 )); then
    log::info "  Stopped containers (${#stopped_containers[@]}):"
    printf '    %s\n' "${stopped_containers[@]}"
    utils::confirm "Remove all stopped containers?" && \
      docker ps -a --filter status=exited -q | xargs docker rm
  fi

  # Dangling images only (untagged, no containers)
  local -a dangling_images
  mapfile -t dangling_images < <(docker images -f dangling=true --format '{{.ID}} {{.Size}}' 2>/dev/null)
  if (( ${#dangling_images[@]} > 0 )); then
    log::info "  Dangling images (${#dangling_images[@]}):"
    printf '    %s\n' "${dangling_images[@]}"
    utils::confirm "Remove dangling images?" && \
      docker images -f dangling=true -q | xargs docker rmi
  fi

  # Builder cache — always safe
  log::info "  Pruning Docker builder cache..."
  safe_rm_cmd docker buildx prune --force
}

devops_reset::_ml_frameworks() {
  local -a ml_paths=(
    "$HOME/.cache/torch"
    "$HOME/.cache/tensorflow"
  )
  # HuggingFace and Ollama — require explicit opt-in due to multi-GB size
  log::warn "ML model caches (.cache/huggingface, .ollama/models) are EXCLUDED by default."
  log::warn "Add them to ~/.config/mac-cleanup/whitelist or use --include-ml-models."

  for p in "${ml_paths[@]}"; do
    [[ -d "$p" ]] || continue
    local size_fmt
    size_fmt=$(utils::format_bytes "$(utils::get_size_bytes "$p")")
    utils::confirm "Delete ${p} (${size_fmt})?" && safe_rm "$p"
  done
}
```

#### B4 — `lib/mail.sh` — Mail & Communication Cleanup

```bash
#!/usr/bin/env bash
# lib/mail.sh — Mail downloads and communication app artifacts (age-gated)

readonly MAIL_ATTACHMENT_AGE_DAYS=30

mail::clean() {
  log::section "Mail & Communications"
  mail::_downloads
  mail::_recent_items
}

mail::_downloads() {
  local -a mail_dirs=(
    "$HOME/Library/Mail Downloads"
    "$HOME/Library/Containers/com.apple.mail/Data/Library/Mail Downloads"
  )
  for dir in "${mail_dirs[@]}"; do
    [[ -d "$dir" ]] || continue
    local size_bytes
    size_bytes=$(utils::get_size_bytes "$dir")
    (( size_bytes < 5242880 )) && continue  # skip if <5 MB
    log::info "Mail downloads: $(utils::format_bytes "$size_bytes") — removing attachments older than ${MAIL_ATTACHMENT_AGE_DAYS} days"
    while IFS= read -r f; do
      safe_rm "$f" "Mail attachment: $(basename "$f")"
    done < <(find "$dir" -type f -mtime "+${MAIL_ATTACHMENT_AGE_DAYS}" 2>/dev/null)
  done
}

mail::_recent_items() {
  # Remove macOS Recent Applications/Documents SFL lists
  local shared="$HOME/Library/Application Support/com.apple.sharedfilelist"
  [[ -d "$shared" ]] || return 0
  for f in "$shared"/com.apple.LSSharedFileList.Recent*.sfl{,2}; do
    [[ -e "$f" ]] && safe_rm "$f" "Recent items: $(basename "$f")"
  done
}
```
